## What this project is

Rebuild of the VAELT marketing site (vaelt.xyz). Single brand — Aurax Digital is fully sunset, do not reference it anywhere in copy, metadata, or code comments. VAELT is a solo-founder web/AI systems studio. Site sells a consultative process (free audit call → custom proposal), not a fixed-price menu.

## Stack

- Next.js 16 (App Router, Turbopack default, Node 20+)
- TypeScript
- Tailwind v4 — tokens live in `app/globals.css` under `@theme`, NOT in `tailwind.config.js`
- `src/` directory structure
- Deploy target: Cloudflare Pages (same as current vaelt.xyz)
- No `next/image` — plain `<img>` with `.webp` assets
- Lenis for smooth scroll (client-side only, respects `prefers-reduced-motion`)

## Hard rules — apply to every page, every component

1. **No price numbers anywhere on the site.** Not in copy, not in structured data, not in a hidden tooltip. Pricing only happens on the call.
2. **No real client names, logos, or links to actually-delivered client sites.** Work page uses clearly-labeled `(concept)` builds only.
3. **Design tokens are locked.** Colors, fonts (Clash Display / Satoshi / VT323), spacing scale — pull from `design-system.md` exactly. Don't introduce new colors or fonts.
4. **Copy comes from `content.md` verbatim.** If something's missing or ambiguous, flag it in `session-log.md` and ask rather than inventing copy.
5. **Never promise "customers."** Use inquiries, leads, visibility, lead flow — this is a copy rule, applies to any new microcopy too (button states, form confirmations, error messages).
6. **Section-tag-row numbering (`[ 01 / 06 ]`) only appears on Home.** It's not a sitewide nav device — see `design-direction.md` for why.

## Session protocol

**At the start of every session:**
1. Read `docs/session-log.md` in full — this is the actual state of the build, not this file.
2. Read `docs/build-guidelines.md` to confirm which phase comes next.
3. Pick up from the last logged checkpoint. Don't re-build what's already marked done.

**At the end of every session:**
1. Append a new entry to `docs/session-log.md` (date, what was built, what's left, any blockers or open questions for Dhanika).
2. Do NOT edit this file (`CLAUDE.md`) to reflect session progress — it stays static. If something here is genuinely wrong or outdated (not just "more got built"), flag it to Dhanika rather than silently editing it.
