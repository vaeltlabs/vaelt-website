## HOW WE BUILD (4-phase panel)

**Section intro:**
- H2: How We Build
- Body: Every project moves through the same four phases — short enough to stay fast, structured enough that nothing gets missed.

**Phase 01 — Discovery**
> We start with a short call, not a questionnaire. What's slowing the business down, what's already working, and what the site or system actually needs to do. Scope and the number get locked before anything starts.

**Phase 02 — Design**
> A direction gets built around the business, not a template. You see it before a single line of code gets written, so there are no surprises at handoff.

**Phase 03 — Build**
> The actual build happens fast — usually days, not weeks — because the scope was locked upfront and the stack stays lean by default.

**Phase 04 — Launch**
> Launch isn't the end. The site or system gets tested on real devices, real connections, and handed over with a care plan if you want one.
